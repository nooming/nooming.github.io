🧩 猫抓扩展保姆级使用教程

本文主要介绍如何使用猫抓浏览器扩展，按照「安装 → 界面 → 抓取 → 下载」这个顺序说明一遍。没有技术基础也可以跟着完成。

如果更习惯看视频：
  • 可以参考这期 B 站教程：电脑必备的浏览器插件：猫抓插件使用教程，安装方法。
  • 视频里有安装演示和 m3u8 视频下载示例，与本文内容基本对应。

一、准备工作：确认你的浏览器

1. 支持的浏览器
  • 推荐使用 Chrome 或 Edge 浏览器。
  • 如果使用的是其他浏览器，建议优先选择 Chrome 或 Edge 安装和使用猫抓。
2. 打开扩展管理页面
  • Chrome：地址栏输入 chrome://extensions/ 后回车。
  • Edge：地址栏输入 edge://extensions/ 后回车。
  • 先大概看一眼这个页面就行，后面本地加载会用到。

二、安装猫抓扩展：扩展商店 & 本地加载

安装猫抓有两种方式：一种是通过扩展商店直接安装，另一种是本地加载源码/压缩包。

方式 A：从扩展商店安装
  • 打开 Chrome 网上应用商店或 Edge 附加组件商店，搜索 「猫抓」或「Cat Catch」。
  • 点进扩展详情页，点击 「添加至 Chrome」/「获取」→「添加扩展」。
  • 等浏览器提示安装完成，在右上角即可看到猫抓图标。
  • 这种方式会自动更新扩展，也不需要手动开启开发者模式。
方式 B：本地加载（用源码安装）
  • 下载猫抓扩展源码压缩包：点击下载 cat-catch.zip（或访问作者的 GitHub 仓库：nooming.github.io 仓库）。
  • 解压这个 ZIP 文件，在解压出来的文件夹里找到 cat-catch 这个文件夹（里面能看到 manifest.json 等文件）。
  • 把这个 cat-catch 文件夹丢到一个不会乱动的位置（桌面临时目录不太推荐）。
  • 回到刚才的扩展管理页面，右上角打开 「开发者模式」。
  • 点击 「加载已解压的扩展」 → 选中刚才的 cat-catch 目录 → 确认。
  • 加载成功后，你会在扩展列表里看到「猫抓」或对应的图标名称，本地版本和商店版用起来是一样的，只是更新需要你自己重新加载。
小提示：
  • 第一次本地加载失败，十有八九是目录选错（没有选到包含 manifest.json 的那一层），重新选一次就好了。
  • 如果你后面想「跟着源码一起学」，本地加载会更方便：改一行代码 → 回到扩展页面点一下「重新加载」，马上生效。

三、认识猫抓的基础界面

安装好后，浏览器右上角会出现一个小猫图标。如果没看到，可以点击扩展按钮，把猫抓「固定」到工具栏。

常见界面模块
  • 主面板：点击猫抓图标弹出的页面，显示当前网页捕获到的各种媒体资源列表。
  • 媒体列表：包括视频、音频、m3u8 流等，每一行通常有名称、类型、大小、操作按钮。
  • 操作按钮：如「下载」「复制链接」「发送到下载器」等。

四、一步一步教你「抓」一个视频

下面用一个典型的视频网页举例，按顺序做即可：

步骤 1：打开目标网页
  • 在浏览器中打开你想要抓的视频页面。
  • 等网页加载完，最好先点一下播放，让视频真正开始加载。
步骤 2：打开猫抓面板
  • 点击浏览器右上角的 猫抓图标。
  • 如果页面资源很多，可以稍等几秒，让扩展把捕获结果整理出来。
步骤 3：识别你要的那一条资源
  • 优先留意 类型为「Media / Video / m3u8」 的条目。
  • 有些站点会有多条清晰度不同的资源（720P / 1080P），可以根据大小/名称大致判断。
  • 看不准时，可以先点「播放」，再回到面板里观察新出现的条目。
步骤 4：选择下载方式
  • 直接下载：点击「下载」按钮，浏览器会直接用默认下载器保存到本地。
  • 复制链接：点击「复制链接」，把真实地址粘贴到 IDM、Aria2 等专业软件下载。
  • m3u8 流：如果是 m3u8，猫抓扩展里通常会有「发送到下载器」或「导出命令」之类的功能，配合脚本/下载器使用。
实用建议：遇到下载速度慢、浏览器容易卡死时，可以优先选择「复制链接 → 用专业下载器（如 IDM、Aria2、Motrix 等）下载」，更稳定。

五、常见问题 & 排查思路

1. 面板里什么都没有？
  • 确认页面的视频已经开始播放过。
  • 尝试刷新页面后，再打开猫抓面板。
  • 某些站点用了非常规协议或强加密，扩展确实可能抓不到，这种情况就不要强求。
2. 下载的文件播放不了？
  • 优先使用 PotPlayer、VLC 这类兼容性更好的播放器。
  • 如果是 m3u8 分片单独下来的，需要先合并/转码，再播放。
3. 只想保存音频怎么办？
  • 有些网站会额外提供音频流，面板里类型会显示为 audio 或类似标识。
  • 也可以先把视频抓下来，再用音频转换工具/转码工具提取音频。

  • 请仅在个人学习、备份等合理场景下使用猫抓扩展。

六、和「浏览器开发者工具」配合使用的思路

简单理解：猫抓更像是「帮你整理好的可视化工具」，而浏览器 Network 面板是「底层原始数据」。

  • 当猫抓已经能识别并列出资源时，直接用猫抓即可。
  • 当遇到猫抓识别不出来的页面，可以配合另一篇「浏览器开发者工具实战」文章，通过 Network 面板手动分析请求。
结语：熟悉一次完整流程之后，后面再遇到类似的视频/音频网页，你会发现：基本都是「打开网页 → 播放 → 打开猫抓 → 选中资源 → 下载」这几步的组合，只是细节上略有差异而已。
